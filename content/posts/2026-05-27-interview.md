## 第一次面试

### 面试时间

2026-04-10 15:00

### 岗位说明

开发实习生（2027 届）

#### 职位描述

工作职责：

- 参与公司编译、持续集成等 DevOps 相关平台的开发、运维工作；
- 参与测试工具平台的开发、运维工作；
- 参与推进相关规范和流程的落地；
- 参与 AI 大模型提升工程效能的调研与实践。

工作要求：

- 2027 届毕业，本科及以上学历，计算机及相关专业；
- 掌握计算机操作系统、计算机网络等基础专业知识；
- 至少熟练掌握一种编程语言（Golang / Java / Python 等），Golang 优先；
- 工作积极主动，思维活跃严谨，具备良好的表达能力、团队协作能力以及较强的抗压能力；
- 【优先】熟悉 Kubernetes 的架构和设计；
- 【优先】具备容器云平台（PaaS）相关实践经验，包括部署、开发、运维等；
- 【优先】熟悉 DevOps 理念；
- 【优先】熟悉 DevOps 常用工具链，如：Git、GitLab、Maven、Gradle、SonarQube 等；
- 【优先】有开源项目经验；
- 【优先】有大模型相关经验。

### 面试过程

这次面试是我和 bilibili 的第一次面试，也是我求职路上的第一次面试。面试官是一位中年程序员老哥，头发比较稀疏，但还挺和蔼的。我记得我们先是从自我介绍开始的，然后聊到了 CI / CD ,我当时还用的是 Gitea + Jenkins + Harbor 的方案 (现在已经迁移到 Forgejo + ArgoCD 了，将镜像仓库和流水线都放在 Forgejo 中)，当时他问我 Jenkins 不会觉得很重吗，我的回答是 Jenkins 的自由度很高，就像一个 Cron plus 一样(现在看来，Jenkins 是该退休了，Forgejo Action 和 GitHub Action 这类才是主流)。紧接着我们我们是聊到了 Vibe Coding，我当时用的 opencode 这个工具，配合 GitHub Copilot 的订阅(届时 GitHub Copilot 还是很香的)，我们聊了最新的几个 LLM 都是有什么特性，比如 Gemini 3.1 pro 前端审美能力比较在线，GPT 5.4 编码能力强等等。聊了很多很多，面试官表示很满意，最后给我出道代码题，但我其实没有重视Leetcode，也没有看什么面经，所以我委婉的表达我这方面能力很弱，面试官也表示理解，说毕竟 AI 时代，面试过程中的算法题的存在意义他也认为不大。  
对了，这次面试的时长我记得是 40 多分钟。

### 面试结果

过了一个星期，还没收到二面的通知。我用微信联系了 HR，她表示面试官对我的能力表示认可，但 HC 有限，这边就没再推进了。

## 第二次面试

### 面试时间

2026-05-27 14:00

### 岗位说明

AI 开发实习生（平台工程方向）【2027 届】

#### 职位描述

工作职责：

- 参与大模型训推平台开发，支撑千卡至万卡训练任务；
- 开发任务调度系统（Kubernetes + Volcano）；
- 集成分布式训练框架（Megatron-LM / DeepSpeed / FSDP）；
- 构建实验管理平台（实验追踪、超参搜索、模型版本管理）；
- 开发 GPU 监控、故障检测、断点续传等运维工具。

工作要求：

- CS 相关专业在读；
- 熟悉 Python，掌握后端框架（FastAPI / Flask / Go 均可）；
- 了解 Kubernetes，有容器化部署经验优先；
- 了解分布式训练原理（DP / MP / PP）优先；
- 有 Linux 系统管理和 Shell 脚本经验；
- 了解 Prometheus / Grafana 监控体系加分；
- 有 MLflow / W&B / Kubeflow 经验加分。

### 面试过程
这次的面试过程很煎熬，虽然从上次面试到现在的这段时间，我学习了很多新东西，进步了很多，但这个岗位好像并不是我想像的那样。面试官说他是干模型训练的，一直在讲各种 GPU 调度，我想和他讲讲我在服务器方面的经验，但他好像不是很感兴趣。尬聊了半小时左右，他表示我更适合去基础架构部门的工作。

### 面试结果
当天晚上，我就看到官网显示初试不通过。
